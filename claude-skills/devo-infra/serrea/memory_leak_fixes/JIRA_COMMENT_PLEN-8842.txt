h2. Summary

*Problem:* OutOfMemoryError causing API 504 errors and service degradation
*Root Cause:* Unbounded caches + missing connection leak detection
*Solution:* Configuration-based bounded caches + leak detection
*Status:* ✅ Successfully deployed to ALL 3 NODES (rolling restart 08:10-08:45 UTC)

h3. 2. Bounded Cache Configuration - BEFORE

{code}
❌ NO CACHE LIMITS
❌ NO EXPIRATION POLICY
❌ UNBOUNDED GROWTH → OutOfMemoryError
{code}

h3. 2. Bounded Cache Configuration - AFTER

{code:properties}
# ✅ BOUNDED CACHES

# Default cache: 10,000 max entries
cache.default.maximumSize=10000
cache.default.expireAfterWrite=3600

# Query cache: 5,000 entries, 30min expiration
cache.query.maximumSize=5000
cache.query.expireAfterWrite=1800

# Session cache: 2,000 entries, 1hr expiration
cache.session.maximumSize=2000
cache.session.expireAfterAccess=3600

# EHCache configuration
ehcache.config.location=/opt/logtrust/serrea/conf/ehcache.xml

# Hibernate cache bounded
hibernate.cache.use_second_level_cache=true
hibernate.cache.use_query_cache=true
hibernate.cache.region.factory_class=org.hibernate.cache.ehcache.EhCacheRegionFactory
{code}

h3. 3. EHCache XML Configuration (NEW FILE)

*File:* {{/opt/logtrust/serrea/conf/ehcache.xml}}

{code:xml}
<ehcache>
    <!-- Default: 10,000 entries, 1hr TTL, LRU eviction -->
    <defaultCache
        maxEntriesLocalHeap="10000"
        timeToLiveSeconds="3600"
        timeToIdleSeconds="1800"
        memoryStoreEvictionPolicy="LRU">
    </defaultCache>

    <!-- Query Cache: 5,000 entries -->
    <cache name="query.results" maxEntriesLocalHeap="5000" />

    <!-- Hibernate Query Cache: 500 entries -->
    <cache name="org.hibernate.cache.internal.StandardQueryCache" maxEntriesLocalHeap="500" />

    <!-- Session Cache: 2,000 entries -->
    <cache name="session.cache" maxEntriesLocalHeap="2000" />

    <!-- User Cache: 1,000 entries -->
    <cache name="user.cache" maxEntriesLocalHeap="1000" />

    <!-- Domain Cache: 500 entries -->
    <cache name="domain.cache" maxEntriesLocalHeap="500" />

    <!-- Alert Cache: 3,000 entries -->
    <cache name="alert.cache" maxEntriesLocalHeap="3000" />

    <!-- Loxcope Query Cache: 2,000 entries -->
    <cache name="loxcope.query.cache" maxEntriesLocalHeap="2000" />
</ehcache>
{code}

*Total Cache Budget:* ~24,000 max entries (bounded)

h2. Key Improvements

{panel:bgColor=#dff0d8|borderColor=#3c763d}
✅ *Leak Detection Enabled*
- Logs warning if connection held > 60 seconds
- Identifies leak location in code for permanent fix

✅ *Bounded Caches*
- Maximum 24,000 cache entries total
- Automatic LRU eviction
- Time-based expiration (30min - 2hr)

✅ *Connection Pool Optimized*
- Pool size increased: 10 → 20 connections
- Active validation with SELECT 1 query
- JMX monitoring enabled

✅ *Automatic Eviction*
- Least Recently Used entries removed first
- Old entries expire automatically
- Memory usage capped
{panel}
